> 먼저 [Wiki](https://en.wikipedia.org/wiki/Candidate_key) 설명을 보시겠습니다. 

> In the relational model of databases, a candidate key of a relation is a minimal superkey for that relation; that is, a set of attributes such that: (1) the relation does not have two distinct tuples (i.e. rows or records in common database language) with the same values for these attributes (which means that the set of attributes is a superkey) (2) there is no proper subset of these attributes for which (1) holds (which means that the set is minimal).

> **(TL;DR)** 아마 minimal이라는 관용적인 표현때문에 혼란이 있을 수 있습니다. [minimal](https://en.wikipedia.org/wiki/Maximal_and_minimal_elements)은 일반적으로 집합 내에 가장 작은 값을 가지는 element를 지칭하는 표현으로 사용됩니다. 그러나 여기서의 minimal은 그러한 관용적 의미가 아니라, (1) 조건을 만족하는 집합 s의 어떠한 진 부분집합도 (1)을 만족할 수 없는 상태를 나타내는 의미로 사용됩니다. 

> (추가) attribute 개수와 관련하여: $A$라는 어트리뷰트가 릴레이션 $r(R)$의 candidate key들 중 하나라고 가정해봅시다. ${B,C}$도 (1)조건과 (2)조건만 만족하면, $r(R)$의 candidate key가 될 수 있습니다. $|{A}| = 1$ 이고 $|{B,C}| =2$ 지만, (1)조건 (2)조건만 만족하면 둘 다 candidate key라고 봅니다. 어트리뷰트 개수와는 무관합니다.

> 이해를 돕기위해 비유적으로 설명드리면, 어떠한 attribute 집합이 (1)조건을 만족할 때, (1)조건 만족을 유지하는 선에서 최대한 다이어트하는 겁니다 (불필요해 보이는 attribute를 제거한다고 보면 됩니다). 다이어트 하다가 이제 다이어트할 수 없는 (즉, 하나라도 빼버리면 (1)조건 유지가 불가능해지는) 상태가 되면, 이제 정말 핵심 attribute들만 남은 것이겠죠. 그러한 attribute 집합은 (1)을 만족하는 minimal한 attribute들이며, 이 경우 candidate key라고 부릅니다. 이해를 돕기위한 informal한 진술이니 공식적인 자리에서 이렇게 답하시면 안됩니다 :)
